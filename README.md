# 🛫 100种不可思议旅行

> 一个由 AI 驱动的旅行内容平台——你只需要输入一个目的地，AI 会自动搜索资料、匹配图片、撰写文案，生成一个完整的旅行体验网页。

---

## ✨ 这个项目做什么？

打开网站，你会看到 5 个精心制作的旅行体验——北极圈极光潜水、亚马逊死藤水仪式、活火山徒步……每一个都像一本小杂志，有故事、有图片、有实用信息。

**最好玩的是**：你可以自己推荐一个想去的地方。输入名字，AI 就会去 Wikipedia 搜资料、去 Pexels 找图片、用大语言模型写文案，几十秒后，一个全新的旅行体验页面就自动生成了。

如果 AI 一时找不到足够的信息（比如太小众的地点），系统会记下来，每天凌晨自动重试，绝不放弃。

---

## 🎯 怎么玩？

```bash
# 1. 启动项目
bash start.sh

# 2. 打开浏览器
# 首页 → http://localhost:5173
# 浏览 5 个初始案例，或者点击「+ 推荐不可思议的体验」

# 3. 输入你想去的地方
# 比如："冰岛蓝冰洞"、"摩洛哥蓝色小镇"、"宫崎骏幽灵公主原址"
# AI 会自动判断真实位置（幽灵公主 → 屋久岛），生成完整页面
```

---

## 🤖 AI 在背后做了什么？

当你在搜索框输入一个目的地后，AI 会依次完成以下工作：

```
你的输入: "宫崎骏幽灵公主原址"
    ↓
🌐 多源搜索    Wikipedia（中文+英文）+ National Geographic
    ↓           AI 发现"幽灵公主"的森林原型是日本屋久岛
📷 智能配图    优先找 Wikipedia 头图 → 没有就搜 Pexels
    ↓           自动下载、提取配色
✍️ 撰写文案    把搜索到的资料喂给大语言模型
    ↓           生成：一句话亮点 + 体验详情 + 背后故事
💾 自动发布    写入数据库 → 你的新页面就上线了！
```

整个过程全自动，你只需要等待几十秒。如果 90 秒内没完成，系统会显示"雷达暂未覆盖"，并在后台继续尝试。

## 📱 页面长什么样？

- **首页** — 全屏轮播背景，5 秒切换，毛玻璃卡片叠加
- **列表页** — 卡片瀑布流，可按类型/景观/评分筛选，无限滚动
- **详情页** — 大图 Hero + Markdown 正文 + 侧边栏 + 同类推荐
- **深色/明亮模式** — 一键切换，米黄色文字在深色背景下温暖舒适
- **移动端适配** — 底部导航栏 + 下拉刷新

## 🔄 内容如何保证质量？

| 环节 | 做了什么 |
|------|---------|
| 搜索 | 中英文 Wikipedia + National Geographic 交叉验证 |
| 图片 | 三级策略：Wikipedia 头图 → Pexels 精确搜 → 景观特征搜 |
| 文案 | DeepSeek 大语言模型生成，禁止夸张营销用语 |
| 地点 | AI 自动修正——"宫崎骏幽灵公主原址"→"屋久岛" |
| 兜底 | 搜不到的记入后台队列，每天凌晨 2 点自动重试，30 天仍失败则人工介入 |

| 做什么 | 用了什么 | 为什么选它 |

## 📋 页面和接口

### 三个页面
| 页面 | 路由 | 内容 |
|------|------|------|
| 首页 | `/` | 全屏轮播 + 开始探索入口 |
| 探索列表 | `/trips` | 筛选 + 卡片网格 + 无限滚动 |
| 体验详情 | `/trips/:id` | 大图 + 故事 + 来源 + 预订 |

### 主要接口
| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/trips` | 获取旅行列表（支持筛选和分页） |
| `GET` | `/api/trips/random` | 随机推荐一个 |
| `GET` | `/api/trips/{id}` | 查看详情 |
| `POST` | `/api/trips` | 提交新目的地 |
| `POST` | `/api/trips/{id}/enrich` | 触发 AI 自动丰富 |
| `GET` | `/api/booking/search` | 生成飞猪预订链接 |
| `GET` | `/api/health` | 健康检查 |

## 🚀 本地运行

需要：**Python 3.12+** 和 **Node.js 20+**

```bash
# 1. 克隆项目
git clone https://github.com/euredachen/100-incredible-trips.git
cd 100-incredible-trips

# 2. 安装后端依赖
cd backend
pip install -r requirements.txt

# 3. 安装前端依赖
cd ../frontend
npm install

# 4. 一键启动
cd ..
bash start.sh
```

打开 `http://localhost:5173` 就能看到网站了。

## 🙋 常见问题

**Q: 为什么我的目的地搜不出来？**
A: 试试用更具体的地名，或者加上英文关键词。如果 90 秒还没出来，系统已经记下了，第二天会自动重试。

**Q: 能加自己的图片吗？**
A: 目前是 AI 自动从 Pexels 搜索配图。后续会支持上传自己的照片。

**Q: 数据存在哪里？**
A: 本地 SQLite 数据库，文件在 `backend/trips.db`。初始有 5 条案例数据。

**Q: 这个项目能部署到线上吗？**
A: 可以。前端构建后是纯静态文件，后端 FastAPI 可部署到任何云服务器。
